Privacy Policy

Effective date: June 2026  ·  Last updated: June 2026

Summary

CyberBench collects no personal data, sends no data to any server, and contains no advertising or tracking of any kind. All operations run entirely on your device.

What data does CyberBench collect?

None. CyberBench does not collect, store, transmit, or share any personal information, usage statistics, crash reports, or analytics data.

Internet access

The app declares the INTERNET permission because certain tools make network connections that you initiate and control:

  • Network Scanner — connects to hosts you specify for port scanning.
  • Domain OSINT — performs DNS lookups and other queries for domains you enter.
  • Web Tools — fetches URLs you provide to inspect HTTP responses.
  • SMB Explorer — connects to SMB servers you configure.

No background connections are made. The app never contacts any server operated by the developer.

Device permissions

PermissionWhy it's needed
INTERNET User-initiated network tools (see above)
CAMERA QR code scanning in the Recipe Builder
ACCESS_FINE_LOCATION Required by Android's Wi-Fi scanning API (Android 9+); no location data is stored or transmitted
ACCESS_WIFI_STATE / CHANGE_WIFI_STATE WiFi Analyzer tool — reads nearby access point info on-device only
NEARBY_WIFI_DEVICES WiFi scanning on Android 13+; replaces location permission for Wi-Fi on newer devices
QUERY_ALL_PACKAGES App Scanner — enumerates installed apps for local security analysis only; package data never leaves the device
POST_NOTIFICATIONS Optional completion notifications for long-running background scans

Local data storage

CyberBench stores data locally on your device only:

  • Saved recipes (local SQLite database)
  • Scan history (local SQLite database)
  • App preferences (SharedPreferences — e.g., screen capture protection, onboarding state)

Cloud backup is disabled (android:allowBackup="false"). This data never leaves your device and is deleted when you uninstall the app.

Third-party services

CyberBench does not integrate any third-party analytics, advertising, crash reporting, or tracking SDKs.

Children's privacy

CyberBench is not directed at children under 13. It is intended for security professionals, students, and researchers conducting authorized security testing.

Authorized use

CyberBench is a security research toolkit. It is intended for testing systems you own or have explicit written permission to test, CTF competitions, and security education. Using these tools against systems without authorization is illegal and unethical.

Changes to this policy

If this policy is updated, the new version will be published at this URL and the effective date will be updated accordingly.
